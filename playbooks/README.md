# playbooks/

Protocolos por dominio. Procedimientos repetibles para situaciones recurrentes.

```
playbooks/
  training/         # ej. progresión-fuerza-básica, deload-protocol, vuelta-tras-lesion
  nutrition/        # ej. batch-cooking-90min, comer-fuera-sin-romper-plan
  meditation/       # ej. respiración-crisis-aguda, ritual-noche
  discipline/       # ej. mala-racha-reset, plan-dia-malo
  finance/          # ej. auditoria-trimestral, regla-72h, cancelacion-suscripciones
  productivity/     # ej. ritual-revision-semanal, time-blocking-realista
  life-strategy/    # ej. decision-irreversible, transicion-profesional
```

## Cuándo crear un playbook

- Cuando una situación se repite ≥3 veces.
- Cuando un protocolo funcionó y queremos no perderlo.
- Cuando hay un patrón que requiere respuesta consistente.

## Formato sugerido

```
# [Nombre del playbook]

## Cuándo aplicarlo
[señales que disparan este playbook]

## Pasos
1. [...]
2. [...]

## Variantes
[según contexto]

## Errores comunes
[...]

## Métrica de éxito
[cómo sabes que funcionó]
```

Los agentes pueden referenciar playbooks de su dominio cuando son aplicables.
